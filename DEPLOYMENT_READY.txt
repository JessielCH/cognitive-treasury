📋 VARIABLES DE ENTORNO (Configurar en Railway)
══════════════════════════════════════════════════════════════════════════════

GOOGLE_API_KEY
  ├─ Obtén en: https://aistudio.google.com/apikey
  └─ Requerido para: Copiloto IA (Gemini)

VITE_API_URL
  ├─ Railway lo genera automáticamente
  ├─ Ej: https://cognitive-treasury-prod.railway.app
  └─ Requerido para: Frontend conecte con Backend

WEIGHTS_STORAGE_PATH
  ├─ Valor por defecto: /app/api/data/rlhf_weights.json
  └─ Nota: Configura un Volumen en Railway para persistencia


🔑 CAMBIOS CLAVE EN EL CÓDIGO
══════════════════════════════════════════════════════════════════════════════

FRONTEND (React)
  Antes:  const response = await fetch("http://localhost:8000/api/chat", ...)
  Después: const API_URL = import.meta.env.VITE_API_URL || "http://localhost:8000"
           const response = await fetch(`${API_URL}/api/chat`, ...)

BACKEND (Python)
  Antes:  WEIGHTS_FILE = "api/rlhf_weights.json"
  Después: WEIGHTS_DIR = os.getenv("WEIGHTS_STORAGE_PATH", os.path.join("api", "data"))
           os.makedirs(WEIGHTS_DIR, exist_ok=True)
           WEIGHTS_FILE = os.path.join(WEIGHTS_DIR, "rlhf_weights.json")

📊 OPCIONES DE PERSISTENCIA EN RAILWAY
══════════════════════════════════════════════════════════════════════════════

OPCIÓN A: Volumen Local (Recomendado para inicio)
  Ventajas:
    ✅ Fácil de configurar
    ✅ Los pesos RLHF persisten entre deploys
  Pasos:
    1. Railway → Settings → Volumes
    2. Crea volumen: /app/api/data
    3. Railway lo monta automáticamente

OPCIÓN B: PostgreSQL (Recomendado a largo plazo)
  Ventajas:
    ✅ Escalable
    ✅ Backups automáticos
    ✅ Queries más complejas
  Pasos:
    1. Railway → New Service → PostgreSQL
    2. Conectar a api/rlhf_engine.py
    3. Usar psycopg2 en lugar de JSON

🆘 TROUBLESHOOTING
══════════════════════════════════════════════════════════════════════════════

¿El backend no levanta?
  → Revisa logs: railway logs
  → Verifica GOOGLE_API_KEY está configurada

¿El frontend no ve el backend?
  → Comprueba VITE_API_URL en Railway
  → Asegúrate que es la URL pública correcta

¿Los pesos RLHF se pierden?
  → Configura un Volumen en Railway
  → Monta en /app/api/data

¿Error de CORS?
  → Backend tiene CORS habilitado (api/index.py)
  → Verifica que frontend está en dominio permitido
